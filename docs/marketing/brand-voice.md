## Core Identity

**Wild Agents** builds Paperclip themes and plugins for developers who care about their tools. Our brand identity is the wolf pack: sharp, coordinated, disciplined, and fast. ## Voice Attributes

### Sharp
Say exactly what you mean. Oblique marketing language erodes trust with technical audiences.

- **Not:** "A beautiful, immersive experience crafted for the modern developer workflow"
- **Yes:** "Dark theme with warm neutrals and high-contrast syntax highlighting"

### Coordinated
Every piece of content — README, release note, forum reply — sounds like it came from the same pack. Consistent terminology, consistent tone, consistent calls to action.

- Always "themes" and "plugins" — never "templates," "extensions," or "add-ons"
- Always "Paperclip" — never "the platform" in first mention
- Always "Wild Agents" as the company name — never "we" without first establishing who we are

### Honest
We never claim capabilities we don't have. If a theme doesn't support dark mode yet, we say so. Technical audiences have high bullshit detectors and long memories.

### Direct
Lead with the thing that matters. Developers scan, they don't read.

- **Not:** "After months of careful research into developer workflow needs and extensive design iteration guided by our creative direction team, we are excited to announce..."
- **Yes:** "New theme: **Umbral**. Dark base, OKLCH grays, designed for 8-hour sessions."

## Messaging Framework

### Primary Message
**Wild Agents builds themes and plugins that respect your workflow.**

### Supporting Messages

**For theme users:**
> Paperclip themes built by developers, for developers. Every token is intentional. Every contrast ratio is tested. Ship with a setup you're not embarrassed by.

**For plugin users:**
> Plugins that extend Paperclip without fighting it. Built on the official SDK. CSS custom properties all the way down.

**For contributors and early adopters:**
> Open-source first. Fork it, extend it, improve it. The pack grows when the work improves.

**For the Paperclip community:**
> We are Paperclip-native. We build with the SDK, on the design token system, with OKLCH all the way. Not against the grain — with it.

---

## Key Differentiators

1. **OKLCH-native design.** We use the full OKLCH color space — perceptual uniformity, wide gamut, no muddy mid-tones. Other themes fake dark mode; ours is engineered.

2. **Agent-built, developer-tested.** Our themes come from a team that runs on Paperclip all day. We notice what hurts. We fix it.

3. **SDK-first, not CSS hacks.** We use `--paperclip-*` CSS custom properties as documented. No `!important` overrides, no DOM patching. Themes upgrade cleanly.

4. **Open, forkable, documented.** Every theme ships with a manifest, documented token list, and deploy script. You can extend it or replace it. We want you to.

5. **Pack quality signal.** Vigia reviews every release. If it doesn't pass QA, it doesn't ship. Small team, high signal.

## Language Guide

### Use
- "theme" / "plugin" — always lowercase unless starting a sentence
- "OKLCH" — always uppercase
- "CSS custom properties" — when being precise; "CSS variables" acceptable in informal context
- "deploy" — not "publish" or "upload"
- "staging" — our Paperclip instance; lowercase
- "the pack" / "Wild Agents" — when referring to the team
- "the Paperclip ecosystem" — when referring to the broader community

### Avoid
- "template," "add-on," "extension" — wrong product category
- "beautiful," "stunning," "amazing" — empty adjectives
- "cutting-edge," "next-generation," "revolutionary" — marketing noise
- "easy to use" — say *what* makes it easy, or don't say it
- "seamless integration" — show it, don't tell it
- Passive voice in release copy — "We shipped X," not "X has been released"

## Tone by Context

| Context | Tone | Example |
|---|---|---|
| Release announcement | Confident, direct, specific | "Umbral v1.1 ships with corrected syntax highlight tokens and a new `--umbral-surface-dim` layer. See diff." |
| README | Technical, welcoming, minimal | Short intro, code block first, link to docs. No paragraphs of backstory. |
| Forum reply | Helpful, peer-to-peer, no sales voice | Answer the question, link to the code. |
| Blog post | Authoritative, demonstrates depth | Show the problem. Show the solution. Show the code. |
| Error / known issue | Honest, specific, actionable | "Token `--paperclip-accent` is missing in v1.0.2. Fix in v1.1 — workaround below." |
| Social post | Sharp, no hashtag spam, one point per post | "New plugin: live token inspector. See your Paperclip CSS variables in real time. Link in bio." |

## Wolf Pack Metaphor — Usage Rules

The wolf pack metaphor is our internal identity anchor. Use it purposefully, not decoratoratively.

**Do use it:**
- In the About/manifesto section of the website or README
- In team introductions and hiring posts
- In brand taglines and promotional copy

**Don't use it:**
- In technical documentation
- In release notes
- As a substitute for substance ("Ship with the pack!" tells nobody anything useful)

The metaphor amplifies — it doesn't replace clarity.
