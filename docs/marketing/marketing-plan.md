# Wild Agents — Marketing Plan v1.0

> Version 1.0 | April 2026 | Owned by Eco (Marketing Lead)

---

## Situation

Wild Agents is a new entrant in the Paperclip ecosystem. We have:
- **Zero audience** — no followers, no installs, no community presence yet
- **Zero budget** — no paid advertising, no sponsored placements
- **High-quality output** — themes engineered with OKLCH, plugins built SDK-first
- **A distinctive identity** — wolf pack brand, technically serious team

The constraint is real: we grow through work, not spend.

---

## Target Audience

### Primary: Paperclip power users
- Developers who run Paperclip daily and notice its default theme limitations
- Likely to explore the plugin/theme ecosystem themselves
- Decision-making horizon: minutes (they try a theme, it works or doesn't)
- Success metric: install and keep

### Secondary: Paperclip developers and integrators
- Teams evaluating Paperclip for internal tooling
- Want reassurance that the ecosystem has momentum
- Decision-making horizon: days to weeks
- Success metric: reference our work in their evaluation

### Tertiary: Developer tooling community at large
- Frontend engineers and design-system practitioners interested in OKLCH
- Not Paperclip users today — potential Paperclip adopters
- Success metric: follows, shares, awareness

---

## Positioning

**For developers using Paperclip who are dissatisfied with the default visual experience:**
Wild Agents themes and plugins are the highest-quality, most technically rigorous options in the ecosystem — because we build them the way we build everything: engineered, tested, open.

**Versus generic themes or DIY CSS:**
- We use the full OKLCH color space; DIY usually means copy-pasted hex codes
- We pass Vigia's QA before every release; no "works on my machine" shipping
- We document everything; no reverse-engineering required

## Marketing Channels (Zero-Budget Stack)

### 1. GitHub (primary distribution + credibility)
**Goal:** Every developer who finds the repo should understand what we are and want to install something.

Actions:
- High-quality README with clear value proposition, code examples, and install instructions
- Detailed release notes on every tag — developers follow releases via GitHub
- `CONTRIBUTING.md` to invite community involvement
- Issue templates to structure user feedback
- Stars and forks as social proof metric

### 2. Paperclip Community Forums / Discord (primary community channel)
**Goal:** Be known as the team that ships good things and helps people.

Actions:
- Introduce Wild Agents when ecosystem forums/channels are identified
- Post release announcements as structured posts, not just links
- Answer questions about Paperclip theming — become the technical authority
- No spam, no self-promotion-only presence. Earn trust first.

### 3. Developer Blog (technical content marketing)
**Goal:** Drive organic discovery via technical content that developers actually want to read.

Topics by priority:
1. "OKLCH for UI developers: why the color space matters for design tokens" (SEO anchor piece)
2. "How Paperclip's CSS custom property system works — and how to extend it"
3. "Building a Paperclip theme from scratch: walkthrough with the Wild Agents SDK"
4. "What good Paperclip QA looks like: our testing checklist"

Distribution: GitHub (docs/blog/), cross-posted to dev.to, Hashnode where appropriate.

### 4. Twitter/X + Mastodon (amplification)
**Goal:** Short-form amplification of releases and content. Secondary to GitHub and community.

Rules:
- One claim per post, specific and true
- No hashtag spam (max 2 per post)
- No promotional threads without content behind them
- Link to actual code or docs, not landing pages

### 5. npm (if we ship JS plugins)
**Goal:** Discovery by developers searching for Paperclip extensions.

Actions:
- Ensure package.json keywords are accurate and specific
- README is the same quality as the GitHub README
- Semantic versioning, reliable release cadence

## Growth Strategy

### Phase 1: Foundation (Weeks 1–4)
Build the infrastructure for growth before trying to drive traffic.

- [ ] README to professional quality
- [ ] First theme shipped and publicly listed
- [ ] Blog post published (at least one technical piece)
- [ ] Community presence established (first intro post in Paperclip community)
- [ ] Basic metrics tracking in place (GitHub stars, fork count, issues opened)

### Phase 2: Momentum (Weeks 5–12)
Build release cadence and community engagement.

- [ ] Monthly releases (new theme or plugin version)
- [ ] Two community content pieces per month (blog posts or forum contributions)
- [ ] Developer feedback loop active (issues, questions answered within 48h)
- [ ] First external mention or reference (another dev mentioning Wild Agents)

### Phase 3: Authority (Month 4+)
Become the reference point for Paperclip theming.

- [ ] Go-to answer for "what themes are available for Paperclip?"
- [ ] Guest contributions or collaborations with other Paperclip community members
- [ ] Metrics review and strategy adjustment

## Constraints

- **No paid spend.** Every acquisition is organic. This means we need quality content and real community value.
- **Small team.** Marketing cadence must be sustainable. One good piece every two weeks beats five rushed tweets.
- **Product is primary.** Nothing in this plan should distract from shipping. Marketing amplifies product — it doesn't substitute for it.
