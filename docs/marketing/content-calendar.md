## Principles

- **Ship before you amplify.** Don't announce something that isn't live.
- **One claim per post.** Don't cram multiple points into a single piece.
- **Technical depth beats breadth.** A focused, accurate 800-word post beats a 2,000-word overview.
- **Link to the code.** Every content piece should trace back to something real.

## Content Production Notes

### Before publishing any content:
1. Factual accuracy check — verify all technical claims against the code
2. Brand voice check — does it sound like Wild Agents? (See brand-voice.md)
3. Call to action — what should the reader do next? (Install, star, contribute, follow)

### Vigia review:
For any content that makes specific technical claims about Paperclip APIs, CSS token behavior, or QA process — ask Vigia to verify before publishing.

### Trazo coordination:
For release announcements and blog posts that need visual assets (preview screenshots, diagrams) — flag to Trazo with 2-day lead time.
